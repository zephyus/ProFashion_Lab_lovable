# 全面自我檢查與修正計畫

目標：用瀏覽器逐一測試網站所有頁面、所有按鈕、所有流程，發現任何錯誤即修正，直到完美。

## 檢查範圍

### 1. 底部導航（每個 tab 點一次）
- 首頁
- 職圖
- 撥號（您撥的號碼是未來）
- 其他既有 tab

### 2. 首頁
- 所有按鈕、連結、卡片點擊
- 滾動與 RWD（手機 viewport）
- 底部導航是否擋住內容

### 3. 職圖頁 `/map`
- 分類篩選（生活產業 / 文化創意 / 技術職人 / 教育陪伴 / 影像媒體）切換
- 地圖上每個職人點點擊
- 下方 carousel 滑動與同步切換
- 職人卡片點擊進入詳情

### 4. 職人詳情頁 `/map/$mentorId`
- 返回按鈕
- 內容區塊顯示（介紹、一天工作、最真實的一面、人格特質、體驗活動）
- 「報名體驗」按鈕

### 5. 報名流程
- 個人預約 → 選時段 → 填表 → 成功頁
- 老師幫班級預約 → 選時段 → 填表（學校 / 班級 / 人數）→ 成功頁
- 表單驗證（必填、錯誤訊息）
- 取消 / 返回各步驟

### 6. 撥號頁 `/call`
- 卡片顯示
- 電話功能（原有功能不動，但要確認沒壞）

### 7. 技術健檢
- Console error / warning
- Network 404 / 500
- TypeScript 編譯（`bunx tsc --noEmit`）
- Route 解析
- 底部導航不擋內容（mobile viewport）

## 執行方式

1. 用 `browser--navigate_to_sandbox` 以 mobile viewport (例：390x844) 開啟
2. 依序走完上述每個流程，每步 `observe` → `act` → `screenshot`
3. 同步監看 `read_console_logs` 與 `list_network_requests`
4. 發現問題即記錄，全部走完後一次批次修正
5. 修正後重新跑一次相關流程驗證
6. 反覆直到 0 error、所有按鈕都有反應、所有流程都能完成

## 交付標準

- Console 無 error
- Network 無 4xx/5xx（除預期外部資源）
- 每個按鈕都有正確反應
- 報名兩種類型都能完整走到成功頁
- TS 編譯通過
- 手機版底部導航不擋內容
