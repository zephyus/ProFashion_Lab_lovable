# 全站按鈕地毯式測試與修正計畫

目標：**親自點擊每一個可見的按鈕、連結、互動元素**，記錄所有異常並修正到完美。

## 測試範圍（每個按鈕逐一點擊）

### 1. 底部導覽（每個分頁都要進入）
- 首頁、職圖、咖啡、配對、來電 — 全部 5 個 tab

### 2. 首頁 `/`
- 所有 hero CTA、卡片、輪播箭頭、分類標籤、頭像、通知鈴鐺、設定齒輪…等**每一顆**

### 3. 職圖 `/map`
- 分類 chip（橫向捲動每一顆都點）
- 地圖每個 pin
- 卡片輪播左右箭頭、每張卡片
- 卡片上的「查看詳情/預約」次按鈕（若有）

### 4. 職人詳情 `/map/$mentorId`
- 返回按鈕、收藏/分享（若有）
- 內容區塊內所有可點元素
- 「報名體驗」浮動按鈕
- 預約 sheet：取消、個人預約、班級預約、上一步、每個時段、確認報名、表單欄位驗證、成功頁的「完成/再看一位」按鈕
- **兩種預約類型都跑完整流程**

### 5. 咖啡 `/cafe`、配對 `/match`、來電 `/call`
- 每張卡片、每顆 CTA、撥號/接聽/掛斷等

### 6. 其他可達路由
- 掃 `src/routes/` 所有檔案，列出未測過的頁面逐一進入測試

## 技術檢查
- Console errors / Network 4xx-5xx 全程監控
- 手機 viewport（390×844）下底部導覽不遮擋內容
- `bunx tsc --noEmit` 0 錯誤

## 執行方式
1. `code--list_dir src/routes` 列出全部路由 → 建立完整按鈕清單
2. 用 browser 工具在 390×844 視窗逐頁 `observe` → 對每個 actionable 元素 `act` → `screenshot`
3. 每發現一個問題 → 記錄 → 批次修正 → 重新驗證直到 0 異常
4. 完成後提供逐頁測試報告（按鈕 / 預期 / 實際 / 結果）

## 交付標準
- 所有按鈕都有「✅ 已點擊並通過」紀錄
- 無 console error、無 4xx/5xx、無 TS 錯誤
- 兩種預約流程跑到成功頁
- 行動裝置版面無遮擋
